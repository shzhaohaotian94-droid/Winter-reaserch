# Winter Research 看板公开部署说明

目标：别人打开一个公网网址即可查看你的看板，而不是访问 `127.0.0.1`。

推荐架构：

- 前端：Vercel，部署 `frontend`
- 后端：Render，部署 `backend`
- 模式：公开只读，别人能看，不能调用 AI，不能改持仓

## 已经准备好的文件

- `render.yaml`：Render 后端部署蓝图
- `backend/Dockerfile`：后端 Docker 启动文件
- `frontend/vercel.json`：Vercel 前端路由配置
- `frontend/.env.example`：前端连接后端的环境变量样例

## 第一步：推到你自己的 GitHub 仓库

不要推到原作者 `simonlin1212/Vibe-Research`。

你需要先在 GitHub 建一个新仓库，例如：

```text
https://github.com/你的用户名/winter-research-dashboard
```

然后在本机仓库里改远端：

```powershell
git remote set-url origin https://github.com/你的用户名/winter-research-dashboard.git
git add .
git commit -m "Prepare public deployment"
git push -u origin main
```

## 第二步：Render 部署后端

1. 打开 Render。
2. New -> Blueprint。
3. 选择你的 GitHub 仓库。
4. Render 会读取根目录的 `render.yaml`。
5. 部署完成后复制后端网址，例如：

```text
https://vibe-research-api.onrender.com
```

默认环境变量：

```text
VR_PUBLIC_READONLY=true
VR_RESEARCH_ROOT=/app/research
VR_ALLOW_ORIGINS=*
```

公开只读模式下，别人不能调用 AI，也不能写入持仓。

## 第三步：Vercel 部署前端

1. 打开 Vercel。
2. Import Git Repository。
3. 选择同一个 GitHub 仓库。
4. Root Directory 选择：

```text
frontend
```

5. Environment Variables 添加：

```text
VITE_API_URL=https://你的-render-后端网址
```

6. Deploy。

## 第四步：分享 Vercel 网址

以后分享 Vercel 给你的网址，例如：

```text
https://winter-research-dashboard.vercel.app
```

不要再分享：

```text
http://127.0.0.1:5899
```

因为 `127.0.0.1` 只代表访问者自己的电脑。

## 注意

- 你的桌面 `思考` 文件夹不会上传。
- 公网版会显示线上研报、板块细分、Serenity 评分等公开数据。
- 如果以后要发布你自己的研报全文，需要先做脱敏，再迁移到云端文件库或数据库。
